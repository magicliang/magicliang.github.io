Maven зҡ„й…ҚзҪ®е’Ңдҫқиө–жҳҜеҚ•ж №з»§жүҝзҡ„

Maven зҡ„жЁЎеқ—з»§жүҝжҳҜж— жі•иҝӣиЎҢеӨҡз»§жүҝзҡ„пјҢеҸӘиғҪдҪҝз”ЁеҚ•ж №з»§жүҝгҖӮ

Maven дёӯ dependencies дёҺ dependencyManagement зҡ„еҢәеҲ«

dependencies еҚідҪҝеңЁеӯҗйЎ№зӣ®дёӯдёҚеҶҷиҜҘдҫқиө–йЎ№пјҢйӮЈд№ҲеӯҗйЎ№зӣ®д»Қз„¶дјҡд»ҺзҲ¶йЎ№зӣ®дёӯз»§жүҝиҜҘдҫқиө–йЎ№пјҲе…ЁйғЁз»§жүҝпјү

dependencyManagement йҮҢеҸӘжҳҜеЈ°жҳҺдҫқиө–е’Ңе®ғд»¬зҡ„зүҲжң¬пјҢе№¶дёҚе®һзҺ°еј•е…ҘпјҢеӣ жӯӨеӯҗйЎ№зӣ®йңҖиҰҒжҳҫзӨәзҡ„еЈ°жҳҺйңҖиҰҒз”Ёзҡ„дҫқиө–гҖӮеҰӮжһңдёҚеңЁеӯҗйЎ№зӣ®дёӯеЈ°жҳҺдҫқиө–пјҢжҳҜдёҚдјҡд»ҺзҲ¶йЎ№зӣ®дёӯз»§жүҝдёӢжқҘзҡ„пјӣеҸӘжңүеңЁеӯҗйЎ№зӣ®дёӯеҶҷдәҶиҜҘдҫқиө–йЎ№пјҢе№¶дё”жІЎжңүжҢҮе®ҡе…·дҪ“зүҲжң¬пјҢжүҚдјҡд»ҺзҲ¶йЎ№зӣ®дёӯз»§жүҝиҜҘйЎ№пјҢе№¶дё” version е’Ң scope йғҪиҜ»еҸ–иҮӘзҲ¶ pom; еҸҰеӨ–еҰӮжһңеӯҗйЎ№зӣ®дёӯжҢҮе®ҡдәҶзүҲжң¬еҸ·пјҢйӮЈд№ҲдјҡдҪҝз”ЁеӯҗйЎ№зӣ®дёӯжҢҮе®ҡзҡ„jarзүҲжң¬гҖӮ

жңүж•Ҳ pom

дёӢиҪҪдҫқиө–дёӯзҡ„жәҗз Ғе’Ңж–ҮжЎЈ

еңЁ maven йҮҢпјҢresolve жңүзү№еҲ«зҡ„еҗ«д№үпјҡд»ІиЈҒгҖӮ

еңЁ .m2 дёӯиҝҷж ·й…ҚзҪ®пјҡ

еңЁйЎ№зӣ®дёӯиҝҷж ·дё»еҠЁй…ҚзҪ®пјҡ

йҷ„иө дёҖдёӘи„ҡжң¬пјҡ

еңЁ ideaдёӯиҝҷж ·й…ҚзҪ®Preference > Build, Execution, Deployment > Build Tools > Maven > importingгҖӮ

bom е’Ң pom

BOM

BOMе®ҡд№ү

BOMпјҲBill of MaterialпјүпјҢзү©ж–ҷжё…еҚ•гҖӮе°ҶжҹҗдёҖдёӘйўҶеҹҹзӣёеә”зҡ„jarз»ҹдёҖз§°д№ӢдёәXXX-BOMдәҲд»Ҙз®ЎзҗҶпјҲдҫӢеҰӮinf-bomгҖҒspring-framework-bomпјүпјҢе№¶дёҚжҳҜMavenзҡ„и§„е®ҡпјҢеҸӘжҳҜдёҖз§ҚзәҰе®ҡдҝ—жҲҗпјҲconvention over configurationпјүгҖӮGradle д№ҹйҮҮз”ЁдәҶиҝҷж ·зҡ„жҖқи·ҜгҖӮ

дёәд»Җд№ҲиҰҒдҪҝз”Ёbom

пјҲ1пјүйҒҝе…ҚејҖеҸ‘еҗҢеӯҰйңҖиҰҒе…іеҝғеҗ„дёӘAPIзҡ„зүҲжң¬е…ізі»пјҡBOMиҝҷз§Қж–№ејҸеңЁ dependencyManagement жҢҮе®ҡдәҶеҗ„дёӘдҫқиө–зҡ„зүҲжң¬пјҢе№¶дёҚдјҡдҪҝеҫ—иҝҷдәӣдҫқиө–е№¶иў«зңҹжӯЈеј•е…ҘпјҲд»…еҒҡзүҲжң¬з®ЎзҗҶдҪҝз”ЁпјүгҖӮеңЁdependencyдёӯиҝҳиҰҒжҢүйңҖеј•е…ҘгҖҒдҪҶзңҒеҺ»дәҶйңҖиҰҒеҲ¶е®ҡversionзҡ„йә»зғҰгҖӮ

пјҲ2пјүдҫҝдәҺеҺҶеҸІзүҲжң¬APIзҡ„ж”¶еҪ’гҖӮ

пјҲ3пјүеҸҜд»ҘиҜҙйҖҗжёҗдҪҝз”Ё bom жқҘз»ҹдёҖз®ЎзҗҶжҹҗдёӘеӣўйҳҹпјҲйўҶеҹҹпјүзҡ„ API зүҲжң¬е·ІжҲҗдёәдёҖз§Қи¶ӢеҠҝгҖӮ

scope зҡ„еҗ«д№ү

scopeе…ғзҙ зҡ„дҪңз”ЁпјҡжҺ§еҲ¶ dependency е…ғзҙ зҡ„дҪҝз”ЁиҢғеӣҙгҖӮйҖҡдҝ—зҡ„и®ІпјҢе°ұжҳҜжҺ§еҲ¶ Jar еҢ…еңЁе“ӘдәӣиҢғеӣҙиў«еҠ иҪҪе’ҢдҪҝз”ЁгҖӮ
scopeе…·дҪ“еҗ«д№үеҰӮдёӢпјҡ

compileпјҲй»ҳи®Өпјү

еҗ«д№үпјҡcompile жҳҜй»ҳи®ӨеҖјпјҢеҰӮжһңжІЎжңүжҢҮе®ҡ scope еҖјпјҢиҜҘе…ғзҙ зҡ„й»ҳи®ӨеҖјдёә compileгҖӮиў«дҫқиө–йЎ№зӣ®йңҖиҰҒеҸӮдёҺеҲ°еҪ“еүҚйЎ№зӣ®зҡ„зј–иҜ‘пјҢжөӢиҜ•пјҢжү“еҢ…пјҢиҝҗиЎҢзӯүйҳ¶ж®өгҖӮжү“еҢ…зҡ„ж—¶еҖҷйҖҡеёёдјҡеҢ…еҗ«иў«дҫқиө–йЎ№зӣ®гҖӮ

provided

еҗ«д№үпјҡиў«дҫқиө–йЎ№зӣ®зҗҶи®әдёҠеҸҜд»ҘеҸӮдёҺзј–иҜ‘гҖҒжөӢиҜ•гҖҒиҝҗиЎҢзӯүйҳ¶ж®өпјҢзӣёеҪ“дәҺcompileпјҢдҪҶжҳҜеҶҚжү“еҢ…йҳ¶ж®өеҒҡдәҶexcludeзҡ„еҠЁдҪңгҖӮ
йҖӮз”ЁеңәжҷҜпјҡдҫӢеҰӮпјҢ еҰӮжһңжҲ‘д»¬еңЁејҖеҸ‘дёҖдёӘweb еә”з”ЁпјҢеңЁзј–иҜ‘ж—¶жҲ‘д»¬йңҖиҰҒдҫқиө– servlet-api.jarпјҢдҪҶжҳҜеңЁиҝҗиЎҢж—¶жҲ‘д»¬дёҚйңҖиҰҒиҜҘ jar еҢ…пјҢеӣ дёәиҝҷдёӘ jar еҢ…е·Із”ұеә”з”ЁжңҚеҠЎеҷЁжҸҗдҫӣпјҢжӯӨж—¶жҲ‘д»¬йңҖиҰҒдҪҝз”Ё provided иҝӣиЎҢиҢғеӣҙдҝ®йҘ°гҖӮ

runtime

еҗ«д№үпјҡиЎЁзӨәиў«дҫқиө–йЎ№зӣ®ж— йңҖеҸӮдёҺйЎ№зӣ®зҡ„зј–иҜ‘пјҢдҪҶжҳҜдјҡеҸӮдёҺеҲ°йЎ№зӣ®зҡ„жөӢиҜ•е’ҢиҝҗиЎҢгҖӮдёҺcompileзӣёжҜ”пјҢиў«дҫқиө–йЎ№зӣ®ж— йңҖеҸӮдёҺйЎ№зӣ®зҡ„зј–иҜ‘гҖӮ
йҖӮз”ЁеңәжҷҜпјҡдҫӢеҰӮпјҢеңЁзј–иҜ‘зҡ„ж—¶еҖҷжҲ‘д»¬дёҚйңҖиҰҒ JDBC API зҡ„ jar еҢ…пјҢиҖҢеңЁиҝҗиЎҢзҡ„ж—¶еҖҷжҲ‘д»¬жүҚйңҖиҰҒ JDBC й©ұеҠЁеҢ…гҖӮ

test

еҗ«д№үпјҡ иЎЁзӨәиў«дҫқиө–йЎ№зӣ®д»…д»…еҸӮдёҺжөӢиҜ•зӣёе…ізҡ„е·ҘдҪңпјҢеҢ…жӢ¬жөӢиҜ•д»Јз Ғзҡ„зј–иҜ‘пјҢжү§иЎҢгҖӮ
йҖӮз”ЁеңәжҷҜпјҡдҫӢеҰӮпјҢJunit жөӢиҜ•гҖӮ

system

еҗ«д№үпјҡsystem е…ғзҙ дёҺ provided е…ғзҙ зұ»дјјпјҢдҪҶжҳҜиў«дҫқиө–йЎ№дёҚдјҡд»Һ maven д»“еә“дёӯжҹҘжүҫпјҢиҖҢжҳҜд»Һжң¬ең°зі»з»ҹдёӯиҺ·еҸ–пјҢsystemPath е…ғзҙ з”ЁдәҺеҲ¶е®ҡжң¬ең°зі»з»ҹдёӯ jar ж–Үд»¶зҡ„и·Ҝеҫ„гҖӮдҫӢеҰӮпјҡ

import

е®ғеҸӘдҪҝз”ЁеңЁдёӯпјҢиЎЁзӨәд»Һе…¶е®ғзҡ„pomдёӯеҜје…Ҙdependencyзҡ„й…ҚзҪ®пјҢдҫӢеҰӮ (BйЎ№зӣ®еҜје…ҘAйЎ№зӣ®дёӯзҡ„еҢ…й…ҚзҪ®)пјҡ

жғіеҝ…еӨ§е®¶еңЁеҒҡSpringBootеә”з”Ёзҡ„ж—¶еҖҷпјҢйғҪдјҡжңүеҰӮдёӢд»Јз Ғпјҡ

з»§жүҝдёҖдёӘзҲ¶жЁЎеқ—пјҢз„¶еҗҺеҶҚеј•е…Ҙзӣёеә”зҡ„дҫқиө–гҖӮ
еҒҮеҰӮиҜҙпјҢжҲ‘дёҚжғіз»§жүҝпјҢжҲ–иҖ…жҲ‘жғіз»§жүҝеӨҡдёӘпјҢжҖҺд№ҲеҒҡпјҹ

жҲ‘д»¬зҹҘйҒ“Mavenзҡ„з»§жүҝе’ҢJavaзҡ„з»§жүҝдёҖж ·пјҢжҳҜж— жі•е®һзҺ°еӨҡйҮҚз»§жүҝзҡ„пјҢеҰӮжһң10дёӘгҖҒ20дёӘз”ҡиҮіжӣҙеӨҡжЁЎеқ—з»§жүҝиҮӘеҗҢдёҖдёӘжЁЎеқ—пјҢйӮЈд№ҲжҢүз…§жҲ‘д»¬д№ӢеүҚзҡ„еҒҡжі•пјҢиҝҷдёӘзҲ¶жЁЎеқ—зҡ„dependencyManagementдјҡеҢ…еҗ«еӨ§йҮҸзҡ„дҫқиө–гҖӮеҰӮжһңдҪ жғіжҠҠиҝҷдәӣдҫқиө–еҲҶзұ»д»Ҙжӣҙжё…жҷ°зҡ„з®ЎзҗҶпјҢйӮЈе°ұдёҚеҸҜиғҪдәҶпјҢimport scopeдҫқиө–иғҪи§ЈеҶіиҝҷдёӘй—®йўҳгҖӮдҪ еҸҜд»ҘжҠҠ dependencyManagement ж”ҫеҲ°еҚ•зӢ¬зҡ„дё“й—Ёз”ЁжқҘз®ЎзҗҶдҫқиө–зҡ„pomдёӯпјҢз„¶еҗҺеңЁйңҖиҰҒдҪҝз”Ёдҫқиө–зҡ„жЁЎеқ—дёӯйҖҡиҝҮimport scopeдҫқиө–пјҢе°ұеҸҜд»Ҙеј•е…Ҙ dependencyManagementгҖӮдҫӢеҰӮеҸҜд»ҘеҶҷиҝҷж ·дёҖдёӘз”ЁдәҺдҫқиө–з®ЎзҗҶзҡ„pomпјҡ

з„¶еҗҺжҲ‘е°ұеҸҜд»ҘйҖҡиҝҮйқһз»§жүҝзҡ„ж–№ејҸжқҘеј•е…Ҙиҝҷж®өдҫқиө–з®ЎзҗҶй…ҚзҪ®

жіЁж„Ҹпјҡimport scopeеҸӘиғҪз”ЁеңЁ dependencyManagement йҮҢйқўгҖӮиҝҷд№ҲеӨҡзҡ„ scope йҮҢйқўпјҢimport д№ҹеӣ жӯӨжҳҜжңҖеҚұйҷ©зҡ„пјҢеӣ дёә import дјҡжҠҠдҫқиө–зӣҙжҺҘеұ•ејҖпјҢиҖҢдёҚжҳҜз”Ёй—ҙжҺҘдј йҖ’зҡ„ж–№ејҸеңЁж–°еә”з”ЁдёӯдҪ“зҺ°пјҢдјҡиҰҶзӣ– parent е’Ң dependencyпјҲеӣ дёәеҜ»ж №и·Ҝеҫ„жңҖзҹӯпјҢй“ҫжҺҘеҷЁдјҡжңҖе…Ҳиў«й“ҫжҺҘдёҠпјүпјҢиҖҢдё”ж— жі•иў« exclude жҺ’йҷӨгҖӮ

иҝҷж ·пјҢзҲ¶жЁЎеқ—зҡ„pomе°ұдјҡйқһеёёе№ІеҮҖпјҢз”ұдё“й—Ёзҡ„packagingдёәpomжқҘз®ЎзҗҶдҫқиө–пјҢд№ҹеҘ‘еҗҲзҡ„йқўеҗ‘еҜ№иұЎи®ҫи®Ўдёӯзҡ„еҚ•дёҖиҒҢиҙЈеҺҹеҲҷгҖӮжӯӨеӨ–пјҢжҲ‘д»¬иҝҳиғҪеӨҹеҲӣе»әеӨҡдёӘиҝҷж ·зҡ„дҫқиө–з®ЎзҗҶpomпјҢд»Ҙжӣҙз»ҶеҢ–зҡ„ж–№ејҸз®ЎзҗҶдҫқиө–гҖӮиҝҷз§ҚеҒҡжі•дёҺйқўеҗ‘еҜ№иұЎи®ҫи®ЎдёӯдҪҝз”Ёз»„еҗҲиҖҢйқһз»§жүҝд№ҹжңүзӮ№зӣёдјјзҡ„е‘ійҒ“гҖӮ

йӮЈд№ҲпјҢеҰӮдҪ•з”ЁиҝҷдёӘж–№жі•жқҘи§ЈеҶіSpringBootзҡ„йӮЈдёӘз»§жүҝй—®йўҳе‘ўпјҹ

й…ҚзҪ®еҰӮдёӢпјҡ

иҝҷж ·й…ҚзҪ®зҡ„иҜқпјҢиҮӘе·ұзҡ„йЎ№зӣ®йҮҢйқўе°ұдёҚйңҖиҰҒз»§жүҝSpringBootзҡ„moduleдәҶпјҢиҖҢеҸҜд»Ҙз»§жүҝиҮӘе·ұйЎ№зӣ®зҡ„moduleдәҶгҖӮ

scopeзҡ„дҫқиө–дј йҖ’

AвҖ“>BвҖ“>CгҖӮеҪ“еүҚйЎ№зӣ®дёәAпјҢAдҫқиө–дәҺBпјҢBдҫқиө–дәҺCгҖӮзҹҘйҒ“BеңЁAйЎ№зӣ®дёӯзҡ„scopeпјҢйӮЈд№ҲжҖҺд№ҲзҹҘйҒ“CеңЁAдёӯзҡ„scopeе‘ўпјҹзӯ”жЎҲжҳҜпјҡ
еҪ“CжҳҜtestжҲ–иҖ…providedж—¶пјҢCзӣҙжҺҘиў«дёўејғпјҢAдёҚдҫқиө–Cпјӣ
еҗҰеҲҷAдҫқиө–CпјҢCзҡ„scopeз»§жүҝдәҺBзҡ„scopeгҖӮ

дҫқиө–зҹ©йҳөиЎЁж ји§ҒпјҡгҖҠIntroduction to the Dependency MechanismгҖӢ

scope дёҺ optional зҡ„еҢәеҲ«

mavenзҡ„ scope еҶіе®ҡдҫқиө–зҡ„еҢ…жҳҜеҗҰеҠ е…Ҙжң¬е·ҘзЁӢзҡ„classpathдёӢгҖӮ- жҹҗдәӣ scope иҝһжң¬йЎ№зӣ®зҡ„ classpath йғҪдјҡиў«еҪұе“ҚгҖӮдҪҝз”Ёжң¬йЎ№зӣ®зҡ„йЎ№зӣ®ж— и®әеҰӮдҪ•йғҪдёҚиғҪз»•иҝҮ scope зҡ„еҪұе“ҚпјҢscope жүҚжҳҜжңҖеҪ»еә•зҡ„еҜ№дј ж’ӯзҡ„йҡ”зҰ»пјҲжҜ”еҰӮ providedпјүгҖӮ

optionalд»…йҷҗеҲ¶дҫқиө–еҢ…зҡ„дј йҖ’жҖ§пјҢдёҚеҪұе“Қдҫқиө–еҢ…зҡ„classpathгҖӮ- дёҚеҪұе“Қжң¬йЎ№зӣ®з”ҹжҲҗзҡ„ jarпјҢеҪұе“ҚдҪҝз”Ёжң¬йЎ№зӣ®зҡ„йЎ№зӣ®гҖӮ

scope дёҺ optional йғҪеҸҜд»Ҙз”ЁйҮҚж–°еЈ°жҳҺдҫқиө–зҡ„ж–№ејҸжқҘеј•е…ҘзјәеӨұдҫқиө–гҖӮ

жҜ”еҰӮдёҖдёӘе·ҘзЁӢдёӯ

A->B, B->C(scope:compile, optional:true)пјҢBзҡ„зј–иҜ‘/иҝҗиЎҢ/жөӢиҜ•classpathйғҪжңүCпјҢAдёӯзҡ„зј–иҜ‘/иҝҗиЎҢ/жөӢиҜ•classpathйғҪдёҚеӯҳеңЁC(е°Ҫз®ЎCзҡ„scopeеЈ°жҳҺдёәcompile)пјҢAи°ғз”ЁBе“Әдәӣдҫқиө–Cзҡ„ж–№жі•е°ұдјҡеҮәй”ҷгҖӮ

гҖҠOptional Dependencies and Dependency ExclusionsгҖӢпјҡ

Optional dependencies are used when itвҖҷs not possible (for whatever
reason) to split a project into sub-modules. The idea is that some of
the dependencies are only used for certain features in the project and
will not be needed if that feature isnвҖҷt used. Ideally, such a feature
would be split into a sub-module that depends on the core
functionality project. This new subproject would have only
non-optional dependencies, since youвҖҷd need them all if you decided to
use the subprojectвҖҷs functionality.

However, since the project cannot be split up (again, for whatever
reason), these dependencies are declared optional. If a user wants to
use functionality related to an optional dependency, they have to
redeclare that optional dependency in their own project. This is not
the clearest way to handle this situation, but both optional
dependencies and dependency exclusions are stop-gap solutions.

optional жҳҜеӨ§йЎ№зӣ®ж— жі•иў«еҲҮеүІжҲҗе°Ҹзҡ„еӯҗжЁЎеқ—зҡ„ж— еҘҲйҖүжӢ©пјҢеҰӮжһңйЎ№зӣ®иҰҒдҪҝз”Ёиў«дҫқиө–жЁЎеқ—зҡ„еҸҜйҖүеҠҹиғҪпјҢеҝ…йЎ»жҳҫејҸең°еҶҚеЈ°жҳҺдёҖйҒҚеҸҜйҖүдҫқиө–пјҢеҗҰеҲҷдјҡдә§з”ҹи°ғз”ЁеҮәй”ҷгҖӮoptional йҳ»ж–ӯдәҶдј йҖ’дҫқиө–гҖӮ
