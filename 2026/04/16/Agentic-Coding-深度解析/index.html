归档说明：原文是一篇 2,519 行的产品与方法论综合文，混合了 Agent Loop、Messages、OpenCode、OMO 命令、Ralph Loop、Subagent、Harness 与多工具配置。稳定知识已经拆入专题文章；产品命令和内部角色等版本事实交由带版本日期的案例文维护。本页保留原 URL。

Agentic Coding 的稳定定义没有变化：模型围绕目标反复选择动作，Harness 执行工具并把观察结果写回工作集，验证器决定任务是否满足完成条件。具体命令、工具数量、内置角色和默认模型会随版本变化，不再放在一篇永久综合文中。

被订正或退役的版本事实

  • Messages 是重要运行态载体，但不是唯一状态。
  • Claude Code 的工具数量、辅助模型和注入位置不能作为跨版本常数。
  • OpenCode、OMO 与 OMC 的命令名、角色名和默认模型必须跟随当前发布版核验。
  • Slash Command、Skill、Rule、Hook 和 Plugin 处在不同生命周期，不能用“都是 Markdown”合并解释。
  • 多 Agent 的收益取决于任务可分性、上下文隔离和合并成本，不由 Agent 数量单独决定。

Agent Teams 为什么有效
