AI 项目配置文件全景图：md 文件的作用与边界

Created2026-03-16|Updated2026-07-17|AI
|Word Count:577|Reading Time:1mins|Post Views:

归档说明：原文按文件名划分 AGENTS.md、CLAUDE.md、rules/ 与 OpenSpec，部分工具归因和优先级已经过时。稳定的时态分类、载体边界和组合方法已迁入新文章；本页保留原 URL 作为迁移索引。

配置文件不宜只按扩展名分类。更可靠的坐标系包含五类组件：内容源、选择器、上下文装配器、执行器和生命周期管理器。AGENTS.md 与 CLAUDE.md 是项目指令载体，Rules 负责作用域选择，Skill 是可调用能力包，OpenSpec 管理需求与变更状态；各工具对目录、优先级和动态加载的支持需要分别核验。

仍然保留的时态分类

  • 存量知识：长期文档、历史决策、案例和术语。
  • 当前对齐：项目指令、架构边界和工作约定。
  • 迭代状态：plan、task、diff、测试结果和阻塞点。
  • 运行时观察：工具输出、日志、浏览器状态和权限结果。

同一份信息会跨时态迁移。计划完成后应沉淀为决策记录或被删除；旧 memory 在源码和生产状态变化后只能作为线索；运行时日志不应直接升级为永久规则。

被订正的旧分类

  • AGENTS.md 不是“多 Agent 专用总指挥”，CLAUDE.md 也不是“单 Agent 专用配置”。
  • Rules 不存在跨产品统一的匹配语义和固定优先级。
  • Skill、Rule、Command 与 OpenSpec 不是同一层级的替代关系。
  • 产品目录示例必须绑定工具、版本和核验日期，不能写成所有项目的标准树。
