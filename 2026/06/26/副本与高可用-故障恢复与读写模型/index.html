ж·ұе…Ҙ ElasticsearchпјҲ11пјүпјҡж•…йҡңжҒўеӨҚдёҺиҜ»еҶҷжЁЎеһӢ

Created2026-06-26|Updated2026-07-22|жҠҖжңҜ
|Word Count:1.9k|Reading Time:7mins|Post Views:

дёҠдёҖзҜҮи§ЈеҶідәҶж•°жҚ®жҖҺж ·еҲҶеёғеҲ° shard дёҠвҖ”вҖ”и·Ҝз”ұе…¬ејҸе’ҢеҲҶзүҮж•°дёҚеҸҜеҸҳзҡ„зәҰжқҹгҖӮиҝҷдёҖзҜҮиҝӣе…Ҙ shard зҡ„еүҜжң¬жңәеҲ¶е’Ңж•…йҡңжҒўеӨҚгҖӮ

жҜҸдёӘ primary shard еҸҜд»Ҙжңүйӣ¶дёӘжҲ–еӨҡдёӘ replica shardгҖӮReplica жҸҗдҫӣдёӨдёӘиғҪеҠӣпјҡй«ҳеҸҜз”ЁпјҲprimary ж•…йҡңж—¶ replica жҺҘз®Ўпјүе’ҢиҜ»жү©еұ•пјҲжҗңзҙўиҜ·жұӮеҸҜд»Ҙе‘Ҫдёӯ replicaпјүгҖӮзҗҶи§Ј primary-replica зҡ„еҗҢжӯҘжЁЎеһӢпјҢжүҚиғҪеңЁдёҖиҮҙжҖ§е’ҢеҸҜз”ЁжҖ§д№Ӣй—ҙеҒҡеҮәжӯЈзЎ®зҡ„й…ҚзҪ®йҖүжӢ©гҖӮ

жң¬ж–ҮеҸӘжҠ“дёҖдёӘй—®йўҳпјҡprimary е’Ң replica д№Ӣй—ҙжҖҺж ·еҗҢжӯҘж•°жҚ®пјҢprimary ж•…йҡңж—¶жҖҺж ·жҒўеӨҚгҖӮ

еҶҷе…ҘжЁЎеһӢ

дёҖжқЎж–ҮжЎЈзҡ„еҶҷе…Ҙи·Ҝеҫ„з»ҸиҝҮ primary shard еҶҚиҪ¬еҸ‘еҲ° replicaпјҡ

иҝҷдёӘиҝҮзЁӢж¶үеҸҠ 4 дёӘи§’иүІзҡ„дәӨдә’пјҢж—¶еәҸдёҠжңүдёҘж јзҡ„е…ҲеҗҺдҫқиө–пјҡ

sequenceDiagram
    participant C as Client
    participant CN as Coordinating Node
    participant P as Primary Shard
    participant R as Replica Shard(s)

    C->>CN: еҶҷе…ҘиҜ·жұӮ
    CN->>P: и·Ҝз”ұеҲ° primary
    P->>P: еҶҷе…Ҙ buffer + translog
    P->>R: е№¶иЎҢиҪ¬еҸ‘еҶҷе…Ҙ
    R->>R: жң¬ең°жү§иЎҢеҶҷе…Ҙ
    R->>P: иҝ”еӣһзЎ®и®Ө
    P->>CN: жүҖжңү in-sync replica е·ІзЎ®и®Ө
    CN->>C: иҝ”еӣһжҲҗеҠҹ

е…ій”®зӮ№еңЁз¬¬ 4-6 жӯҘпјҡprimary еҶҷе…ҘжҲҗеҠҹеҗҺе№¶иЎҢиҪ¬еҸ‘з»ҷжүҖжңү in-sync replicaпјҢзӯүе…ЁйғЁзЎ®и®ӨеҗҺжүҚеҗ‘дёҠиҝ”еӣһгҖӮд»»дҪ•дёҖдёӘ in-sync replica и¶…ж—¶жңӘе“Қеә”пјҢmaster дјҡе°Ҷе®ғд»ҺеҗҢжӯҘйӣҶеҗҲдёӯз§»йҷӨпјҢиҖҢдёҚжҳҜж— йҷҗзӯүеҫ…гҖӮ

иҝҷжҳҜеҗҢжӯҘеӨҚеҲ¶жЁЎеһӢвҖ”вҖ”primary зӯүеҫ…жүҖжңү in-sync replica зЎ®и®ӨеҗҺжүҚиҝ”еӣһжҲҗеҠҹгҖӮwait_for_active_shards еҸӮж•°еҸҜд»ҘжҺ§еҲ¶йңҖиҰҒеӨҡе°‘дёӘжҙ»и·ғ shard зЎ®и®ӨжүҚз®—еҶҷе…ҘжҲҗеҠҹпјҡ

еҖј еҗ«д№ү
1пјҲй»ҳи®Өпјү еҸӘиҰҒ primary еҶҷе…ҘжҲҗеҠҹе°ұиҝ”еӣһпјҲreplica ејӮжӯҘпјү
all жүҖжңү shardпјҲprimary + жүҖжңү replicaпјүйғҪзЎ®и®Ө
NпјҲж•°еӯ—пјү иҮіе°‘ N дёӘ shard зЎ®и®Ө

жіЁж„Ҹй»ҳи®ӨеҖј 1 ж„Ҹе‘ізқҖеҸӘзӯү primary зЎ®и®ӨгҖӮReplica зҡ„еҶҷе…ҘжҳҜеңЁ primary иҝ”еӣһжҲҗеҠҹд№ӢеҗҺе№¶иЎҢиҝӣиЎҢзҡ„вҖ”вҖ”еҰӮжһң primary еңЁиҪ¬еҸ‘еҲ° replica д№ӢеүҚеҙ©жәғпјҢиҝҷжқЎж•°жҚ®еҸҜиғҪдёўеӨұгҖӮи®ҫ wait_for_active_shards: all еҸҜд»ҘдҝқиҜҒжүҖжңү replica йғҪеҶҷе…ҘжҲҗеҠҹпјҢдҪҶе»¶иҝҹжӣҙй«ҳгҖӮ

In-Sync Allocation IDs

ES з»ҙжҠӨдёҖдёӘ in-sync allocation IDs йӣҶеҗҲпјҢи®°еҪ•е“Әдәӣ shard еүҜжң¬е’Ң primary дҝқжҢҒеҗҢжӯҘгҖӮиҝҷдёӘжҰӮеҝөзұ»дјј Kafka зҡ„ ISRпјҲIn-Sync ReplicasпјүгҖӮ

еҪ“дёҖдёӘ replica иҗҪеҗҺеӨӘеӨҡпјҲжҜ”еҰӮзҪ‘з»ңеҲҶеҢәгҖҒиҠӮзӮ№йҮҚеҗҜпјүпјҢmaster дјҡжҠҠе®ғд»Һ in-sync йӣҶеҗҲдёӯз§»йҷӨгҖӮиў«з§»йҷӨзҡ„ replica дёҚеҶҚжҺҘж”¶еҶҷе…ҘиҜ·жұӮпјҢйңҖиҰҒеҒҡ recovery жүҚиғҪйҮҚж–°еҠ е…ҘгҖӮ

иҜ»еҸ–жЁЎеһӢ

жҗңзҙўиҜ·жұӮеҸҜд»Ҙе‘Ҫдёӯ primary жҲ–д»»дҪ• in-sync replicaгҖӮES дҪҝз”Ё adaptive replica selectionпјҲARSпјүиҮӘеҠЁйҖүжӢ©е»¶иҝҹжңҖдҪҺзҡ„еүҜжң¬пјҡ

ARS з»јеҗҲиҖғиҷ‘иҠӮзӮ№зҡ„жҗңзҙўйҳҹеҲ—й•ҝеәҰгҖҒеҺҶеҸІе“Қеә”ж—¶й—ҙгҖҒиҠӮзӮ№иҙҹиҪҪжқҘеҒҡйҖүжӢ©гҖӮиҝҷж„Ҹе‘ізқҖжҗңзҙўиҙҹиҪҪиҮӘеҠЁеңЁ primary е’Ң replica д№Ӣй—ҙиҙҹиҪҪеқҮиЎЎгҖӮ

Replica ж•°йҮҸи¶ҠеӨҡпјҢжҗңзҙўеҗһеҗҗи¶Ҡй«ҳпјҲжӣҙеӨҡеүҜжң¬еҸҜд»ҘжңҚеҠЎжҗңзҙўиҜ·жұӮпјүпјҢдҪҶеҶҷе…Ҙеҗһеҗҗи¶ҠдҪҺпјҲжҜҸж¬ЎеҶҷе…ҘйңҖиҰҒеҗҢжӯҘеҲ°жӣҙеӨҡеүҜжң¬пјүгҖӮ

ж•…йҡңжҒўеӨҚ

Primary ж•…йҡң

еҪ“ primary shard жүҖеңЁиҠӮзӮ№ж•…йҡңж—¶пјҡ

д»Һ primary ж•…йҡңеҲ°жҒўеӨҚе®ҢжҲҗпјҢж•ҙдёӘйӣҶзҫӨз»ҸеҺҶдёҖж¬ЎзҠ¶жҖҒиҝҒз§»пјҡ

stateDiagram-v2
    [*] --> Normal: йӣҶзҫӨжӯЈеёёиҝҗиЎҢ
    Normal --> Detected: primary жүҖеңЁиҠӮзӮ№ж•…йҡң
    Detected --> Promoting: master йҖүжӢ© in-sync replica
    Promoting --> NewPrimary: replica жҸҗеҚҮдёә primary
    NewPrimary --> Serving: ж–° primary жҺҘж”¶иҜ»еҶҷ
    Serving --> Recovery: ж•…йҡңиҠӮзӮ№жҒўеӨҚдёҠзәҝ
    Recovery --> Syncing: ж—§ primary дҪңдёә replica иҝҪиө¶ж•°жҚ®
    Syncing --> Normal: recovery е®ҢжҲҗпјҢйҮҚж–°еҠ е…Ҙ in-sync йӣҶеҗҲ

жіЁж„Ҹ Promoting еҲ° NewPrimary иҝҷдёҖжӯҘвҖ”вҖ”master еҸӘд»Һ in-sync йӣҶеҗҲдёӯйҖүжӢ©еҖҷйҖүиҖ…пјҢиҗҪеҗҺеӨӘеӨҡзҡ„ replica дёҚдјҡиў«жҸҗеҚҮпјҢйҒҝе…Қж•°жҚ®еӣһйҖҖгҖӮж•…йҡңиҠӮзӮ№жҒўеӨҚеҗҺи§’иүІеҸҚиҪ¬пјҡеҺҹжқҘзҡ„ primary еҸҳжҲҗ replicaпјҢйҖҡиҝҮ peer recovery иҝҪиө¶ж–° primary зҡ„ж•°жҚ®гҖӮ

Shard Recovery

еҪ“дёҖдёӘ shard йңҖиҰҒжҒўеӨҚпјҲиҠӮзӮ№йҮҚеҗҜгҖҒreplica йҮҚж–°еҠ е…Ҙпјүж—¶пјҢES жү§иЎҢ peer recoveryвҖ”вҖ”д»Һ primary еӨҚеҲ¶ж•°жҚ®еҲ°йңҖиҰҒжҒўеӨҚзҡ„ replicaпјҡ

Recovery жңҹй—ҙ primary з»§з»ӯжӯЈеёёжңҚеҠЎгҖӮRecovery е®ҢжҲҗеҗҺпјҢжҒўеӨҚзҡ„ shard еҠ е…Ҙ in-sync йӣҶеҗҲпјҢејҖе§ӢжҺҘж”¶ж–°зҡ„еҶҷе…Ҙе’ҢжҗңзҙўиҜ·жұӮгҖӮ

Unassigned Shards

еҪ“ shard ж— жі•иў«еҲҶй…ҚеҲ°д»»дҪ•иҠӮзӮ№ж—¶пјҢзҠ¶жҖҒеҸҳдёә UNASSIGNEDгҖӮеёёи§ҒеҺҹеӣ пјҡ

  • иҠӮзӮ№ж•°дёҚи¶іпјҲreplica дёҚиғҪе’Ң primary еңЁеҗҢдёҖиҠӮзӮ№пјү
  • зЈҒзӣҳж°ҙдҪҚзәҝи¶…ж Ү
  • allocation filtering жҺ’йҷӨдәҶжүҖжңүеҸҜз”ЁиҠӮзӮ№

з”Ё _cluster/allocation/explain е®ҡдҪҚеҺҹеӣ пјҡ

е®һйӘҢпјҡи§ӮеҜҹеүҜжң¬иЎҢдёә

жЁЎејҸжҸҗзӮјпјҡдё»д»ҺеӨҚеҲ¶ + еҗҢжӯҘзЎ®и®Ө

з»ҙеәҰ еҗҢжӯҘеӨҚеҲ¶ ејӮжӯҘеӨҚеҲ¶
жҢҒд№…жҖ§ й«ҳпјҲжүҖжңүеүҜжң¬зЎ®и®Өпјү дҪҺпјҲеҸҜиғҪдёўеӨұжңҖиҝ‘еҶҷе…Ҙпјү
еҶҷе…Ҙе»¶иҝҹ й«ҳпјҲзӯүеҫ…жңҖж…ўзҡ„еүҜжң¬пјү дҪҺ
йҖӮз”ЁеңәжҷҜ йҮ‘иһҚж•°жҚ®гҖҒдёҚеҸҜдёўеӨұ ж—Ҙеҝ—гҖҒеҸҜйҮҚж”ҫ
ES й…ҚзҪ® wait_for_active_shards: all wait_for_active_shards: 1пјҲй»ҳи®Өпјү

е·ҘзЁӢиҝҒз§»иЎЁ

жҰӮеҝө Elasticsearch Kafka MySQL MongoDB
еӨҚеҲ¶еҚ•е…ғ Shard (primary/replica) Partition (leader/follower) Database (master/slave) ReplicaSet (primary/secondary)
еҗҢжӯҘжңәеҲ¶ иҪ¬еҸ‘еҶҷе…Ҙж“ҚдҪң Log replication Binlog replication Oplog replication
еҗҢжӯҘйӣҶеҗҲ In-sync allocation IDs ISR (In-Sync Replicas) Semi-sync replicas Majority write concern
иҜ»еҸ–и·Ҝз”ұ Adaptive replica selection Consumer fetches from leader/follower иҜ»еҶҷеҲҶзҰ»дёӯй—ҙд»¶ Read preference
ж•…йҡңеҲҮжҚў Master жҸҗеҚҮ replica Controller йҖүж–° leader MHA / orchestrator иҮӘеҠЁйҖүдёҫ

еёёи§ҒиҜҜи§Ј

иҜҜи§ЈдёҖпјҡreplica и¶ҠеӨҡж•°жҚ®и¶Ҡе®үе…ЁгҖӮ Replica ж•°йҮҸеўһеҠ жҸҗеҚҮдәҶиҜ»еҗһеҗҗе’Ңе®№й”ҷиғҪеҠӣпјҢдҪҶжҜҸдёӘ replica йғҪйңҖиҰҒеӯҳеӮЁе®Ңж•ҙзҡ„ shard ж•°жҚ®гҖӮ3 дёӘ replica ж„Ҹе‘ізқҖ 4 еҖҚеӯҳеӮЁгҖӮдё”еҶҷе…Ҙе»¶иҝҹйҡҸ replica ж•°еўһеҠ гҖӮйҖҡеёё 1-2 дёӘ replica е°ұи¶іеӨҹгҖӮ

иҜҜи§ЈдәҢпјҡreplica е’Ң primary зҡ„ж•°жҚ®е®Ңе…Ёе®һж—¶дёҖиҮҙгҖӮ й»ҳи®Ө wait_for_active_shards: 1 ж—¶пјҢеҶҷе…Ҙиҝ”еӣһеҗҺ replica еҸҜиғҪиҝҳжІЎе®ҢжҲҗеҗҢжӯҘгҖӮжһҒзҹӯж—¶й—ҙзӘ—еҸЈеҶ…жҗңзҙў replica еҸҜиғҪзңӢдёҚеҲ°жңҖж–°еҶҷе…ҘгҖӮ

иҜҜи§ЈдёүпјҡеҚ•иҠӮзӮ№йӣҶзҫӨи®ҫзҪ® replica жңүж„Ҹд№үгҖӮ еҚ•иҠӮзӮ№йӣҶзҫӨдёӯ replica ж— жі•еҲҶй…ҚпјҲдёҚиғҪе’Ң primary еңЁеҗҢдёҖиҠӮзӮ№пјүпјҢйӣҶзҫӨзҠ¶жҖҒе§Ӣз»ҲжҳҜ yellowгҖӮеҚ•иҠӮзӮ№еә”иҜҘи®ҫ number_of_replicas: 0гҖӮ

з»ғд№ 

  1. еңЁеҚ•иҠӮзӮ№йӣҶзҫӨдёӯеҲӣе»әдёҖдёӘ 1 replica зҡ„ indexпјҢз”Ё _cluster/allocation/explain жҹҘзңӢ replica ж— жі•еҲҶй…Қзҡ„еҺҹеӣ гҖӮз„¶еҗҺи®ҫ replica дёә 0пјҢи§ӮеҜҹйӣҶзҫӨзҠ¶жҖҒеҸҳдёә greenгҖӮ

  2. з”Ё _cat/shards?v жҹҘзңӢ shard еҲҶеёғпјҢзҗҶи§Ј prirepпјҲp=primary, r=replicaпјүгҖҒstateгҖҒnode еӯ—ж®өзҡ„еҗ«д№үгҖӮ

  3. жҹҘзңӢ _cat/recovery дәҶи§ЈжӯЈеңЁиҝӣиЎҢжҲ–жңҖиҝ‘е®ҢжҲҗзҡ„ recovery ж“ҚдҪңгҖӮ

зі»еҲ—еҜјиҲӘ

дёҠдёҖзҜҮ дёӢдёҖзҜҮ
еҲҶзүҮдёҺи·Ҝз”ұпјҡж•°жҚ®еҲҶеёғзҡ„ж ёеҝғжңәеҲ¶ йӣҶзҫӨеҚҸи°ғпјҡMaster йҖүдёҫдёҺйӣҶзҫӨзҠ¶жҖҒеҗҢжӯҘ
