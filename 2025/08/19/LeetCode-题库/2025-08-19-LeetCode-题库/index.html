分析二分查找的一个技巧是：不要出现 else，而是把所有情况用 else if 写清楚，这样可以清楚地展现所有细节。

right]，或者带个具体的数字进去 [3, 2]，可见这时候区间为空，因为没有数字既大于等于 3 又小于等于 2 的吧。所以这时候
while 循环终止是正确的，直接返回 -1 即可。

while(left < right) 的终止条件是 left == right，写成区间的形式就是 [right,
right]，或者带个具体的数字进去 [2, 2]，这时候区间非空，还有一个数 2，但此时 while 循环终止了。也就是说区间 [2,
2] 被漏掉了，索引 2 没有被搜索，如果这时候直接返回 -1 就是错误的。

转成十进制的求和

官方解法2

一些基础的位运算知识：

  • 异或运算 (^) 模拟无进位加法-意味着不同返回1，而同返回0：

    • 0 ^ 0 = 0(0 + 0 = 0, 无进位)
    • 1 ^ 0 = 1(1 + 0 = 1, 无进位)
    • 0 ^ 1 = 1(0 + 1 = 1, 无进位)
    • 1 ^ 1 = 0(1 + 1 = 10, 无进位部分是 0)

  • 只有当两个位都是 1 时（即1 & 1 = 1），相加才会产生进位，这个比较难理解。这句话的意思是，如果a和b在某一位（比如最低位）都是，则1 ^ 1 = 0了，实际上这进位（1+1等于2，应该进位，变成 10）去哪里了呢？要靠1 & 1 = 1得知进位。所以这两个操作要一起搭配

  • 迭代过程：

    • 将 a ^ b 的结果看作是当前步骤的和（不包含进位）。
    • 将 (a & b) << 1 的结果看作是当前步骤产生的进位。

现在问题变成了：将“当前步骤的和”与“当前步骤的进位”相加。

  • “异或无进位，与后左移一”
    • 异或无进位：用异或 (XOR) 运算计算不考虑进位的加法结果。
    • 与后左移一：用与 (AND) 运算找出进位，然后将结果左移一位（或乘以2）。

java 对大二进制数的支持并不好。

java.math.BigInteger answer = x.xor(y); ... java.math.BigInteger carry = x.and(y).multiply(java.math.BigInteger.valueOf(2));

  • 这两行代码实现了一步不带循环的二进制加法模拟：

    • 第一行计算不考虑进位的和 (answer)。
    • 第二行计算产生的进位 (carry)。

  • 举例：假设x = 3(二进制11), y = 1(二进制1)。

    • 计算answer(无进位和):
      • x.xor(y) -> 3 ^ 1
      • 二进制: 11 ^ 01 = 10
      • 十进制: 3 ^ 1 = 2
      • 所以 answer = 2(二进制10)。这表示11 + 01在不考虑进位时的结果是10。
    • 计算carry(进位):
      • x.and(y) ->3 & 1
      • 二进制: 11 & 01 = 01
      • 十进制: 3 & 1 = 1
      • 这个结果1(二进制01) 表示，在最低位 (第0位) 产生了进位。
      • multiply(BigInteger.valueOf(2))->1 * 2
      • 二进制:01 << 1 = 10(将进位移到它应该去的高一位)
      • 十进制:1 * 2 = 2
      • 所以carry = 2(二进制10)。这表示实际的进位值是10。

  • 下一步：算法现在需要计算answer + carry，也就是2 + 2。这又会变成计算(2 ^ 2)和((2 & 2) << 1)...直到carry变成 0。

141. 环形链表

142. 环形链表2

数学理论：

假设：

  • 链表头到环入口的距离为 a
  • 环入口到快慢指针相遇点的距离为 b
  • 相遇点到环入口的距离为 c（环的剩余部分）

那么：

  1. 慢指针走过的距离：a + b
  2. 快指针走过的距离：a + b + c + b = a + 2b + c
  3. 因为快指针速度是慢指针的 2 倍，所以：2(a + b) = a + 2b + c
  4. 化简得：a = c

这意味着：从链表头到环入口的距离等于从相遇点到环入口的距离。

这个算法实际上一种寻找边界算法，等价于 leftbound查找，因为left此时的值为大于等于 target 的最小的右索引。

要死记的两个结论：

  1. 如果用双闭的方式结尾，最后肯定是：right + 1 = left，得到不存在区间-虽然不变式的性质还是没变：right 指向最后一个 < target 的位置；left 指向第一个>=target的位置。此时的l是r的上一个节点，也是最后一个有效的（>= target）的r的节点，所以这个节点满足>=的特性。
  2. 如果是用左闭右开的方式结尾，最后肯定是：left=right。nums[left] 可能越界，别去试；插入位置是 left，直接返回最稳妥。right 不是一个“数据位置”，而是一个“边界标记”。r 的收缩保证了 r 右侧都 >= target，而 l 的右移保证了 l 左侧都 < target，最终 l 落在第一个 >= target 的位置。。
  3. 不管是开区间right还是闭区间right，l 是第一个满足 >= target 的位置，所以不管开闭区间，l都是target的潜在区间的左边界。当l的值合法，target存在的时候，l既可以当理论左边界使用，也可以当实际左边界使用。

  • “l 永远是第一个 >= target 的位置”-所以开闭区间左边界算法都确认l合法即返回。

  • “闭区间结束 l == r+1，开区间结束 l == r”

  • “r 是边界，l 是位置；l 左都小，r 右都大”

  • “左边界看 l，右边界看 l-1（但不是同一个 l）”

  • “右边界 = 第一个大于 target 的位置 - 1”

  • “找右边界，就要往右走一步，再退回来”

  • “l 是插入点，l-1 是右边界候选”

  • “闭区间用 r = mid - 1，开区间用 r = mid”
