最佳实践建议

  1. 使用具体版本标签：避免使用 latest
  2. 合并 RUN 指令：减少镜像层数
  3. 利用构建缓存：将变化少的指令放在前面
  4. 使用 .dockerignore：排除不必要的文件
  5. 最小化镜像体积：使用 alpine 或 slim 版本
  6. 非 root 用户运行：提高安全性

六、存储卷管理

Volume 类型对比

Docker 提供三种主要的存储方式：

  1. Volume（卷）：由 Docker 管理，存储在 /var/lib/docker/volumes/
  2. Bind Mount（绑定挂载）：将宿主机文件或目录挂载到容器
  3. tmpfs：存储在内存中，容器停止后数据丢失
